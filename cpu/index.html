▯ INPUT FORMAT

Configure processes using these parameters:

  • ID — Process identifier (P1, P2, etc.)
  • Arrival Time — When process enters ready queue
  • Burst Time — CPU time required (must be > 0)
  • Priority — Used only in Priority Scheduling (lower = higher)
▯ ALGORITHMS EXPLAINED

FCFS - First Come First Serve

Non-preemptive algorithm. Processes execute in arrival order. Simple but prone to convoy effect. O(n) complexity.

✓ Best for: Batch systems, simple scheduling

SJF - Shortest Job First

Non-preemptive algorithm. Selects process with smallest burst time. Minimizes average waiting time. O(n log n) complexity.

✓ Best for: Minimizing average waiting time

PRIORITY SCHEDULING

Non-preemptive algorithm. Processes assigned priorities. Lower number = higher priority. Can cause starvation. O(n²) complexity.

✓ Best for: Real-time systems, OS kernel processes

ROUND ROBIN (RR)

Preemptive algorithm. Each process gets fixed time quantum. Fair to all processes. O(n) complexity.

✓ Best for: Time-sharing systems, responsive systems
▯ FORMULAS
  • Completion Time (CT) = Time when process finishes execution
  • Turnaround Time (TAT) = CT − Arrival Time
  • Waiting Time (WT) = TAT − Burst Time
  • CPU Utilization = (Total Busy Time ÷ Total Time) × 100%
